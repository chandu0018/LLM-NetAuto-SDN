## 2. Technical Blueprint of the System

### A. Core Architecture & Components
1. **Natural Language Intent Control Layer**: 
   - Uses a local LLM client (Ollama/Llama-3) integrated via LangChain LCEL (LangChain Expression Language) to translate high-level natural language user inputs (e.g., *"Allow TCP traffic from host h1 to host h2 with high priority"*) into structured JSON OpenFlow rules or ONOS Intent configurations.
   - Includes **Deterministic Fallback Parsing** to handle system timeouts or critical management intents (like *"Remove all custom flow rules"*) instantly without LLM latency overhead.
2. **Retrieval-Augmented Generation (RAG) System**:
   - Leverages a vector database (ChromaDB) to index and retrieve active network topologies, historical intents, system logs, and remediation policies.
   - Enables semantic retrieval of similar past intents to speed up LLM translation, guarantee translation consistency, and maintain historical auditability.
3. **Real-Time Telemetry & Monitoring Layer**:
   - A multi-threaded background telemetry collector that continuously polls port statistics (bytes, packets, dropped packets, errors) from active network switches (via ONOS REST API or high-fidelity simulation engine).
   - Computes real-time rate deltas (MB/s throughput, packet rate, packet drop percentages) from counter values.
4. **Machine Learning Anomaly Detection Layer**:
   - Employs an online unsupervised ML model (`IsolationForest`) pre-seeded with baseline Gaussian-distributed normal traffic profiles to allow instant "trained" operation on system boot.
   - Features **Signature-Based False-Positive Filtering**: Normal statistical fluctuations flagged by Isolation Forest as `"unknown"` anomalies are automatically discarded. True anomalies must match mathematical thresholds corresponding to active attack signatures:
     - **DDoS Simulator**: Huge asymmetric rx-to-tx ratio spikes.
     - **Traffic Spike / Bandwidth Hog**: Throughput exceeding 50 MB/s.
     - **Packet Drop Attack**: Port dropped packet rates exceeding 5%.
5. **Closed-Loop Feedback & Self-Healing Engine**:
   - If a true ML anomaly is confirmed, the feedback loop triggers.
   - It queries the RAG system for past resolved cases, generates a natural language remediation action, translates it through the LLM parser into OpenFlow blocking/rate-limiting rules, and automatically deploys the rules to the switches via the controller API to instantly isolate or mitigate the threat.
6. **Unified Management UI**:
   - A clean, state-of-the-art FastAPI backend combined with a Streamlit front-end featuring dynamic network topology visualization, active flow tables, live telemetry charts, real-time alert logs, and system health status.

### B. Tech Stack
* **SDN Controller**: ONOS (Open Network Operating System) REST API / high-fidelity Python Simulator.
* **LLM Core**: Local Ollama (Llama-3 / Mistral) with LangChain LCEL.
* **Vector DB**: ChromaDB.
* **Telemetry & ML**: Python, Scikit-learn (`IsolationForest`), Pandas, NumPy.
* **Dashboard**: Streamlit, FastAPI, Uvicorn, Requests.
