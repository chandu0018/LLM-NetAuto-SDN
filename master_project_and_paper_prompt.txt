### B. Core Functional Blocks
1. **Intent-Control Translation**: Translates natural language network constraints (e.g. *"Allow TCP traffic from h1 to h2 with priority 40000"*) to OpenFlow flow rules using local LLMs (Ollama/Llama-3) via LangChain. A Regex Fallback catches management rules (like *"Clear all flows"*) immediately without LLM timeout overheads.
2. **ChromaDB RAG Engine**: Indexes device and link descriptions. When a user requests a configuration, it retrieves similar historical intents and injects them into the LLM prompt, removing translation hallucinations.
3. **Telemetry & Feature Engineering**: A polling daemon computes delta-based statistics over a 5-second interval:
   - $Throughput = \Delta Bytes / \Delta t$ (Scaled to MB/s)
   - $DropRate = \Delta Packets_{dropped} / \Delta Packets_{rx}$
4. **Online ML Isolation Forest & Signature Filter**: Flags outliers based on 5 features (`bytes_rx_rate`, `bytes_tx_rate`, `packets_rx_rate`, `drop_rate`, `error_rate`). Flags that do not match specific signatures (DDoS, bandwidth hogs, traffic spikes, packet drop attacks) are automatically classified as false-positives and dropped.
5. **Closed-Loop Self-Healing**: When a true anomaly signature is detected (e.g., DDoS), the system automatically searches ChromaDB for a remediation plan, builds a blocking intent via LLM, and instantly deploys it to mitigate the threat.
